# 📊 Sales Data Analysis using Python

## 📌 Project Overview

This project was developed as part of my **Summer Internship**. The main objective of this project is to analyze the **April 2019 Sales Dataset** using Python and generate meaningful business insights through data cleaning, exploratory data analysis (EDA), and data visualization.

The project demonstrates the complete data analysis workflow, starting from loading raw data to generating visual reports that help understand sales performance and customer purchasing behavior.

---

# 🎯 Objectives

The objectives of this project are:

- Load and understand the sales dataset.
- Clean and preprocess the raw data.
- Handle missing values and duplicate records.
- Convert data into appropriate data types.
- Create new features for analysis.
- Analyze sales performance.
- Identify best-selling products.
- Compare sales across different cities.
- Identify peak sales hours.
- Visualize the results using charts.
- Generate meaningful business insights.

| Column | Description |
|---------|-------------|
| Order ID | Unique order number |
| Product | Product purchased |
| Quantity Ordered | Number of products purchased |

# ⚠️ Challenges Faced

During the development of this project, several technical challenges were encountered. Solving these issues helped improve my practical understanding of Python, data analysis, and version control.

### 1. Python Installation

Initially, Python was not recognized in Command Prompt and VS Code due to configuration issues. This was resolved by correctly installing Python and updating the system environment.

### 2. Library Installation

While installing the required Python libraries, an installation error occurred due to a missing Jupyter-related file. After troubleshooting, the required libraries were installed successfully.

### 3. Dataset Loading

The project initially failed because the dataset was not placed inside the correct project folder. Organizing the folder structure correctly solved the issue.

### 4. File Path Errors

A **FileNotFoundError** occurred because the dataset path was incorrect. This issue was resolved by verifying the project directory and updating the file path.

### 5. Empty Dataset Error

An **EmptyDataError** occurred because an incorrect or empty CSV file was being used. Downloading the correct dataset resolved the problem.

### 6. Data Cleaning

The dataset contained missing values, duplicate records, repeated header rows, and incorrect data types. These issues were resolved before performing any analysis.

### 7. Feature Engineering

Additional columns such as **Sales**, **Month**, **Hour**, and **City** were created to support meaningful analysis and visualization.

### 8. Graph Generation

Initially, only a few graphs were generated because the complete program had not been executed. Running the full script successfully generated all visualization charts.

### 9. Git Installation

While uploading the project to GitHub, Git was not installed on the system. Installing Git for Windows resolved the issue.

### 10. Git Configuration

Git initially failed to create commits because the username and email were not configured. Configuring Git using `git config --global` solved the problem.

### 11. Uploading to GitHub

Learning the Git workflow—including repository initialization, staging files, committing changes, connecting to GitHub, and pushing the project—was an important part of the project experience.
